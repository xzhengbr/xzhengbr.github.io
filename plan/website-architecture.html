个人网站架构与工作流程图
Hugo + GitHub Pages · 从本地编写到用户访问的完整数据流
① 本地开发环境 · Local Machine (macOS)
📝
内容文件
content/*.md
用 Markdown 格式写文章、日志、项目介绍。纯文本文件，无需了解 HTML。
内容层 — 你的核心输出物
推荐配置
文件路径：content/posts/
格式：Front Matter (YAML) + Markdown
编辑器：VS Code + Markdown Preview
🎨
Hugo 源码
themes/ · layouts/ · config.toml
Hugo 项目结构：主题控制样式，layouts 定制模板，config.toml 设置站点参数。
内容管理层 — 负责把 .md 转换为 HTML
推荐配置
主题：PaperMod / Congo / Blowfish
安装：brew install hugo
预览：hugo server -D
⚡
Hugo 构建引擎
hugo build → public/
执行 hugo 命令后，在毫秒内将所有 .md 文件编译成完整的静态 HTML/CSS/JS 文件包，输出到 public/ 目录。
编译器 — 生成最终可发布的静态文件
推荐配置
不需手动运行；由 GitHub Actions 自动触发
构建时间：通常 < 1 秒
git push origin main
开发者推送源码（含 .md 文章）到远程仓库
② GitHub · 代码仓库 + 自动化构建
🗄️
GitHub Repository
github.com/user/blog
存储所有 Hugo 源文件（.md 内容、主题、模板），是项目的"版本控制中心"与"构建触发器"。
版本控制 + 协作 + 历史记录
推荐配置
可见性：Public（GitHub Pages 免费托管需公开）
分支策略：main → 触发自动部署
免费
🤖
GitHub Actions
.github/workflows/deploy.yml
监听 main 分支推送事件，自动执行：安装 Hugo → 运行 hugo build → 将 public/ 部署到 gh-pages 分支。
CI/CD 自动化管道 — 免去手动部署
推荐配置
使用官方 Hugo action：peaceiris/actions-hugo
触发条件：on: push: branches: [main]
免费（每月 2000 分钟）
🌐
GitHub Pages
user.github.io/blog
GitHub 内置的静态文件托管服务。直接读取 gh-pages 分支的文件并对外提供 HTTP 访问，默认提供 HTTPS。
托管平台 — 存储并对外提供静态网页
推荐配置
Source：Deploy from branch → gh-pages
自定义域名：在 Settings → Pages 填入
免费 · HTTPS 自动开启
绑定自定义域名（可选）
将购买的域名指向 GitHub Pages 服务器 IP
③ 域名 & DNS
🏷️
域名注册商
yourname.com
购买你的专属域名。域名本身只是一个"门牌号"，需要通过 DNS 记录才能指向实际服务器。
唯一标识 — 你在互联网上的地址
推荐注册商
Cloudflare Registrar（最便宜，无溢价）
Namecheap · Google Domains
费用：约 ¥60–100/年
📡
DNS 记录配置
CNAME / A Record
在 DNS 控制台添加记录，将域名映射到 GitHub Pages 的 IP 地址或 CNAME 地址，告诉互联网"访问这个域名请去哪个服务器"。
地址翻译 — 把域名转化为服务器 IP
推荐配置
根域名：A @ → 185.199.108.153（GitHub Pages IP × 4条）
子域名：CNAME www → user.github.io
TTL：3600
④ CDN 加速（推荐）
☁️
Cloudflare CDN
边缘节点缓存
将域名 DNS 托管给 Cloudflare 后，它自动在全球 300+ 节点缓存你的静态文件，显著提升国际访问速度（包括香港）。
加速层 — 降低延迟、防 DDoS、管理 SSL
推荐配置
SSL 模式：Full（Strict）
缓存规则：Cache Everything for static assets
免费计划即可满足个人站需求
💡
为什么需要 CDN？
GitHub Pages 服务器在美国。香港 / 大陆直连延迟可达 200–400ms。接入 Cloudflare 后，静态文件从最近节点响应，延迟降至 20–50ms。
此层可选，但强烈推荐
HTTP 请求 → 返回 HTML/CSS/JS 文件
用户浏览器通过域名访问，获取页面文件
⑤ 用户访问
🔍
DNS 解析
域名 → IP 地址
用户在浏览器输入 yourname.com，本地 DNS 查询将域名解析为 Cloudflare 或 GitHub Pages 的 IP 地址。
📦
页面加载
浏览器渲染 HTML
浏览器收到 HTML 文件，依次加载 CSS、JS、图片，渲染出完整页面。全程无需服务器计算，响应极快。
👤
最终用户
任意设备 · 全球访问
用户看到你的个人网站，浏览文章、作品集、联系方式。网站 7×24 在线，无需你做任何操作。
